\section{Predictive Modeling}
% ---------------------------------------------------------------------

\subsection{Experimental Design}

The predictive task is framed as a regression problem, with the daily Polymarket Trump win probability as a continuous dependent variable ranging between 0 and 1. A standard random cross-validation approach is inappropriate here (Cerqueria et al. 2020): shuffling observations would allow the model to train on future data to predict the past, introducing look-ahead bias and producing overly optimistic performance estimates that would not hold in deployment (Makridakis et al., 2018). Instead, an expanding-window walk-forward cross-validation scheme (Tashman, 2000) was adopted, illustrated in (Figure \ref{fig:pred_walk_forward_cv}).

The full dataset spans July 5 to November 4, 2024 (123 daily observations), split into 109 train/validation days, a one-day gap, and a 14-day held-out test set (October 22 -- November 4). Three expanding folds are constructed within the train/validation portion, each growing the training window forward in time while the validation set immediately follows. The one-day gap at each fold boundary prevents rolling-feature bleed.

To provide a meaningful benchmark, a naïve baseline was defined as predicting zero daily change. This serves as the minimum bar for all reported results.

\subsection{Feature Engineering}

Features were constructed from eight distinct sources and aggregated to a daily observation, yielding one row per calendar day over the full study period. A complete overview of all 196 engineered features is provided in Table~\ref{tab:feature_overview} in the Appendix.

\subsection{Data Preparation}

Missing values were handled through deterministic, forward-looking-safe imputation rules: financial columns were forward-filled to account for market closures, domain-specific leading gaps were filled with neutral priors, and all remaining missing values were forward-filled then zero-filled.

Feature selection was then applied exclusively on the train/validation portion of the data (109 observations) to avoid leaking any test-set information into the feature mask, constituting a mild limitation discussed later. Three sequential filters were applied: a variance threshold (var < 1e-5) to remove near-constant features, a Spearman correlation filter ($|\rho| < 0.10$ with the target) to discard features with negligible monotonic relationship to the Polymarket odds, and a pairwise correlation filter ($|\rho| > 0.90$) to remove redundant features, retaining in each correlated pair the feature with the higher absolute Spearman correlation with the target. These three filters reduced the feature set from 196 to 112 features, which formed the input to all predictive models. The feature numbers of these features are underlined in Table~\ref{tab:feature_overview}.

\subsection{Modeling}

Four regression algorithms were trained and evaluated across eight feature sets of increasing complexity: Ridge regression, Random Forest, Support Vector Machine (SVM), and XGBoost. The eight feature sets are described in Tables~\ref{tab:feature_sets} and~\ref{tab:feature_overview} in the Appendix.

\subsection{Performance Evaluation}

Model performance was assessed on the 14-day held-out test set using four metrics: Mean Absolute Error (MAE), Root Mean Squared Error (RMSE), Directional Accuracy (DA), and R². DA was treated as the primary metric, with MAE as a tiebreaker. DA measures the proportion of days on which the model correctly predicts the direction of movement in the Polymarket odds (Pesaran \& Timmermann, 1992), making it the operationally most meaningful criterion in this context. A full overview of test-set results is provided in Table~\ref{tab:results} and Table~\ref{tab:results_2} in the Appendix.

13 of 32 model--feature set combinations achieved DA = 0.5714 (8/14 test days correct), outperforming the naïve baseline (DA = 0.000, MAE = 0.0162) and a random coin flip. The best overall was SVM on the Full (111f) set, with DA = 0.5714 and the lowest MAE of 0.0143 (Figure \ref{fig:pred_svm_result}). Strong DA was not limited to the full set: the minimal Lag (2f) specification also reached DA = 0.5714 with XGBoost and SVM, suggesting much of the directional signal is already captured by autoregressive features. The Financial (10f) set likewise achieved DA = 0.5714 with SVM, indicating market indicators carry some meaningful predictive information. Adding social and traditional media features did not consistently improve DA over these simpler baselines, although the Full set did yield the best MAE.

To understand the driving forces behind model predictions, SHAP values were computed for the best-performing model on the Full (111f) feature set using KernelExplainer (Figure \ref{fig:pred_svm_shap} and Figure \ref{fig:pred_beeswarm}). The absolute SHAP values are very small across all features, reflecting the difficulty of the prediction task: no single feature exerts a dominant, consistent influence. Nonetheless, social media engagement and emotion features from Bluesky and Reddit feature most prominently, with financial features such as oil\_return\_1d playing a secondary role.
