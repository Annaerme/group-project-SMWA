\section{Predictive Modeling}
% ---------------------------------------------------------------------

\subsection{Experimental Design}

The predictive task is framed as a regression problem, with the daily Polymarket Trump win probability as a continuous dependent variable ranging between 0 and 1. A standard random cross-validation approach is inappropriate here: shuffling observations would allow the model to train on future data to predict the past, introducing look-ahead bias and producing overly optimistic performance estimates that would not hold in deployment \citep{makridakis2018}. Instead, an expanding-window walk-forward cross-validation scheme was adopted, illustrated in Figure~X.

The full dataset spans July~5 to November~4, 2024 (123 daily observations), split into 109 train/validation days, a one-day gap, and a 14-day held-out test set (October~22 -- November~4). Three expanding folds are constructed within the train/validation portion, each growing the training window forward in time while the validation set immediately follows. The one-day gap at each fold boundary prevents rolling-feature bleed, mirroring the one-day-ahead prediction task.

To provide a meaningful benchmark, a na\"ive baseline was defined as predicting zero daily change (yesterday's odds = today's odds). Any model that fails to outperform this baseline offers no genuine predictive value beyond simple persistence, and it therefore serves as the minimum bar for all reported results.

\subsection{Feature Engineering}

Features were constructed from eight distinct sources and aggregated to a daily observation, yielding one row per calendar day over the full study period. A complete overview of all 196 engineered features is provided in Table~X in the Appendix.

For each social media source (Bluesky and Reddit), posts were classified into three buzz groups (\textit{TrumpBuzz}, \textit{HarrisBuzz}, and \textit{ElectionBuzz}) via keyword matching. Daily volume, engagement, and sentiment features were computed per group, using RoBERTa for Bluesky and VADER for Reddit, supplemented by eight NRCLex emotion scores. News articles from Media Cloud were similarly classified by buzz group and political leaning, with VADER sentiment aggregated per leaning. Google Trends provided daily search interest for five election-related keywords, alongside derived gap and share features. Polling averages were forward-filled on days without new data, with rolling averages and momentum features added. Five financial market indicators were included alongside derived return and volatility features. Calendar features capture temporal dynamics such as days remaining until election day and days elapsed since the last major political event, where the set of key events was defined by the authors based on their assessment of the most significant moments of the campaign. Finally, autoregressive features were added to allow models to capture momentum in the odds series.

\subsection{Data Preparation}

Missing values were handled through a set of deterministic, forward-looking-safe imputation rules. Financial columns were forward-filled to account for market closures on weekends and public holidays. The \texttt{days\_since\_last\_event} variable was filled with 30 for the first eight days of the period (July~5--12), preceding the first coded event, as a neutral ``long ago'' prior. Rolling poll features with leading missing values were filled with zero, reflecting no observed movement. All remaining missing values, arising from lag columns and rolling windows at the start of the series, were forward-filled and subsequently zero-filled where no prior value was available.

Feature selection was then applied exclusively on the train/validation portion of the data (109 observations) to avoid leaking any test-set information into the feature mask, constituting a mild limitation discussed further in Section~X. Three sequential filters were applied: a variance threshold ($\text{var} < 10^{-5}$) to remove near-constant features, a Spearman correlation filter ($|\rho| < 0.10$ with the target) to discard features with negligible monotonic relationship to the Polymarket odds, and a pairwise correlation filter ($|\rho| > 0.90$) to remove redundant features, retaining in each correlated pair the feature with the higher absolute Spearman correlation with the target. Spearman rank correlation was preferred over Pearson given the bounded, skewed, and non-normal nature of many features. These three filters reduced the feature set from 196 to 112 features, which formed the input to all predictive models.

\subsection{Modeling}

Four regression algorithms were trained and evaluated across eight feature sets of increasing complexity: Ridge regression, Random Forest, Support Vector Machine (SVM), and XGBoost. The eight feature sets range from a minimal autoregressive specification using only two Polymarket probability lags, to a full feature set combining all 111 post-selection features. Intermediate sets isolate specific source groups as well as combinations thereof, allowing a structured assessment of the marginal contribution of each data source. A complete overview of all feature sets and hyperparameter search grids is provided in Tables~X and~Y in the Appendix.

Each model was tuned via grid search over the three walk-forward CV folds, selecting the combination that minimised mean CV MAE. Ridge and SVM were trained on standardised features, with the \texttt{StandardScaler} fitted on the training fold only. Random Forest and XGBoost are scale-invariant and were trained on raw features.
